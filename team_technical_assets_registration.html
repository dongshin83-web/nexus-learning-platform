Registration flow

등록 순서를 확인하세요

모든 Asset type은 같은 4단계를 따르며, 선택한 유형 안에서 단계별 상세 기준과 예시를 펼쳐 확인합니다.

4 STEPS
  1. 1 AI와 대화하기 선택한 Asset type의 질문 순서에 따라 경험과 판단을 설명합니다. 실제 명칭·수치·파일·링크는 Placeholder와 방향성 표현으로 바꿉니다. 권장 AI 도구 유료 AI 계정이 있다면 — 현재 사용 중인 ChatGPT, Gemini, Copilot 등의 유료 계정을 그대로 활용하세요. 유료 AI 계정이 없다면 — 무료 Gemini를 우선 권장합니다. 음성 또는 텍스트 답변부터 JSON 생성까지 같은 대화에서 이어갈 수 있습니다. 공통 권고: AI가 만든 요약을 먼저 확인한 뒤 최종 JSON 생성을 요청하고, 코드 블록과 다운로드 파일의 내용이 같은지 확인하세요.
  2. 2 반입용 JSON 파일 만들기 같은 AI 대화창에 유형별 Prompt를 붙여 넣고, AI 답변에 첨부된 Handoff .json 파일을 다운로드합니다.
  3. 3JSON 확인·사내 반입하기다운로드한 .json 파일 또는 코드 블록으로 직접 저장한 UTF-8 파일을 검토한 뒤 승인된 방식으로 사내 등록 화면에 가져옵니다.
  4. 4사내 Library에 직접 등록하기실제 용어·근거·원본 링크와 관계를 보완한 뒤 같은 화면에서 DB 저장과 Reviewer 검토를 요청합니다. 등록 JSON을 다시 다운로드하지 않습니다.

확정 원칙 음성 파일을 별도로 전사하지 않습니다. 음성 또는 텍스트로 설명한 동일한 대화창에서 JSON까지 생성하고, 실제 정보는 외부 대화가 아니라 사내 등록 화면에서만 복원합니다.

1 · 요청 배경과 판단 질문 AI 질문

어떤 상황에서 요청이 발생했고, 요청자는 언제까지 무엇을 결정해야 했나요?

좋은 답변 예시

[사업부]가 [신규 제품]의 상세 설계를 확정하기 전에, 반복 충격 조건에서 [연결부] 손상 가능성이 낮은 구조를 선택해야 했습니다. 시제품 평가 전에 [설계안 A]와 [설계안 B]의 상대 위험을 비교해 달라는 요청이었습니다.

2 · Simulation 대응 AI 질문

두 설계안을 어떤 조건과 관점으로 비교했나요?

좋은 답변 예시

두 설계안에 동일한 재료 모델과 구속 조건을 적용하고, 반복 충격 시 [연결부]의 변형 집중 위치와 하중 전달 경로를 비교했습니다. 절대 수치의 합격·불합격보다 두 안의 상대 차이와 취약 위치가 반복 조건에서도 일관적인지를 확인했습니다.

3 · 판단 근거 AI 질문

요청자가 설계안을 선택할 수 있도록 어떤 판단 근거를 전달했나요?

좋은 답변 예시

[설계안 B]는 기존 취약 위치의 변형 집중을 상대적으로 낮추고 하중을 주변 구조로 분산했습니다. 다만 새로운 모서리 부위에 국부 집중이 나타나, B안을 바로 확정하기보다 해당 부위의 형상을 보완한 뒤 평가하는 것이 안전하다고 설명했습니다.

4 · 요청자 피드백 AI 질문

요청자는 결과를 어떻게 받아들였고 어떤 후속 요청을 했나요?

좋은 답변 예시

회의에서 취약 위치와 원인을 함께 설명한 점이 설계안 선택에 도움이 됐다는 피드백을 받았습니다. 요청자는 [설계안 B]의 국부 집중을 줄일 수 있는 보완 형상 두 가지를 추가로 비교해 달라고 요청했습니다.

5 · 실제 영향 AI 질문

Simulation 결과 이후 실제 판단이나 다음 행동이 어떻게 달라졌나요?

좋은 답변 예시

[설계안 A]는 다음 단계 적용 후보에서 제외됐습니다. [설계안 B]는 취약 부위를 보완한 조건으로 변경됐고, 변경안이 다음 시제품 평가 대상에 반영됐습니다. 평가 항목에도 Simulation에서 확인한 두 취약 위치가 우선 점검 부위로 추가됐습니다.

6 · 적용범위와 한계 AI 질문

이 판단을 어떤 상황에 재사용할 수 있고, 그대로 적용하면 안 되는 조건은 무엇인가요?

좋은 답변 예시

유사한 연결 구조와 동일한 종류의 충격 조건에서 설계안의 상대 위험을 비교할 때 재사용할 수 있습니다. 재료 구성, 체결 방식 또는 구속 조건이 달라지면 하중 전달 경로가 바뀔 수 있으므로 이번 결과의 절대값과 우선순위를 그대로 적용하면 안 됩니다.

1 · 기술 GapAI 질문
기존 방식으로 해결하기 어려웠던 반복 문제는 무엇이었나요?

                        
좋은 답변 예시
유사한 [신규 구조]에서 신뢰성 위험 검토가 반복됐지만, 기존 모델은 평가의 손상 위치를 일관되게 재현하지 못했습니다. 과제마다 조건을 조정하는 방식으로는 공통 판단 기준을 만들기 어려웠습니다.

                        
2 · 목표와 검증AI 질문
무엇을 판단할 수 있게 만드는 것이 목표였고 어떻게 검증했나요?

                        
좋은 답변 예시
평가 전에 설계안의 상대 위험과 취약 위치를 구분하는 것을 목표로 했습니다. 기존 평가 결과를 기준으로 주요 입력 인자를 단계적으로 바꾸고, 손상 위치와 설계안 위험 순서가 여러 조건에서 반복되는지 확인했습니다.

                        
3 · 확보 기술AI 질문
과제를 통해 무엇을 알게 됐고 어떤 판단이 가능해졌나요?

                        
좋은 답변 예시
[접촉 조건]이 취약 위치를 결정하는 주요 인자임을 확인하고 표준 입력 기준을 정립했습니다. 유사 구조에서는 평가 전에 설계안의 상대 위험을 비교할 수 있지만, 재료 구성이나 체결 방식이 달라지면 추가 검증이 필요합니다.

                        
4 · 경영성과 기여AI 질문
확보한 기술이 개발·시험·품질 또는 의사결정에 어떤 기여를 했나요?

                        
좋은 답변 예시
평가 전에 위험이 높은 후보를 제외할 수 있게 되어 불필요한 시제품 검토와 후기 설계 변경 가능성을 줄였습니다. 구체적인 비용 절감액은 확인되지 않았지만, 설계 후보를 조기에 축소할 판단 근거를 확보했습니다.

                        
5 · 프로세스 변화AI 질문
