Version Scope v1: 검색/필터 + 내부 링크 허브

자산 Library

7개 기술영역 Map

기술영역은 기본 분류로 두고, AI/타 Domain/신뢰성/공정 연계는 보조 태그로 겹쳐서 봅니다.

함께 키워가는 성장 Loop

기술자산화는 한 번 등록하고 끝나는 문서함이 아니라, 대표 사례를 계속 읽고 보완하는 팀 학습 흐름입니다.

01대표 사례 선별
02모델/조건/판단 기준 분해
03파트 내 리뷰와 보완
04내부 링크와 교육 포인트 연결
05유사 과제에서 재사용

프로토타입 수정 기준

세부 자료는 회사 내부 시스템 링크로 연결하고, 샘플 데이터는 team_technical_assets.js 상단에서 직접 수정합니다.

1. 카드 하나가 하나의 자산 단위

BP, 모델, 물성/경계조건, 교육자료를 같은 라이브러리에서 검색하되, 유형 필드로 구분합니다.

2. 기술영역은 하나, 연계 태그는 여러 개

예: 충격 사례에 AI 연계, 신뢰성 연계, 타 Domain 연계를 함께 달아 교차 탐색할 수 있게 합니다.

3. 링크는 내부망에서 교체

현재 링크는 placeholder입니다. 내부 클라우드 또는 문서 시스템 주소로 교체하면 됩니다.

4. 성과보다 재사용 질문을 먼저 작성

“무엇을 해석했는가”보다 “다음 사람이 무엇을 판단하고 재사용할 수 있는가”를 먼저 적습니다.
