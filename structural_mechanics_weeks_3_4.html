12-Week Plan

3~4주차의 목표는 Abaqus/CAE를 능숙하게 조작하는 것이 아니라, 팀원이 만든 선형 정적 모델의 구조를 읽고 결과가 의사결정에 쓸 수 있는지 판단하는 것입니다. 화면의 contour plot보다 먼저 `.inp`, `.odb`, `.msg`, `.sta`, reaction balance, field/history output을 읽는 습관을 만듭니다.

Abaqus Workflow Map

Abaqus/CAE의 각 module은 단순 메뉴가 아니라 해석 가정이 쌓이는 순서입니다. 팀장은 버튼을 외우기보다 각 단계에서 어떤 물리적 가정이 고정되는지 봐야 합니다.

1. Part

형상과 이상화 수준

해석 대상의 geometry를 정의합니다. 이 단계에서 이미 3D solid, shell midsurface, beam idealization 중 무엇을 볼지 결정됩니다.

  • 팀장 질문: 제거한 fillet, hole, fastener detail이 stress path를 바꾸는가?
  • 팀장 질문: 관심 결과가 global stiffness인가, local stress인가?
2. Property

재료와 section

재료 물성, section, 두께, beam profile을 부여합니다. 결과의 절대값은 대부분 여기서 이미 강하게 제한됩니다.

  • 선형 탄성에서는 E와 nu의 출처와 온도 범위를 확인합니다.
  • shell/beam은 section orientation과 두께 정의를 반드시 확인합니다.
3. Assembly

부품 관계와 좌표계

part instance를 배치하고 전체 구조 좌표계를 구성합니다. 이후 하중, 구속, 결과 component의 방향 해석이 여기와 연결됩니다.

  • 좌표계가 결과 component S11/U1의 의미를 바꿉니다.
  • instance dependency와 part-level mesh 관리 방식을 확인합니다.
4. Step

해석 절차

이번 구간에서는 Static, General을 중심으로 봅니다. step은 단순 계산 버튼이 아니라 어떤 평형 문제를 푸는지 정의합니다.

  • 선형 정적이면 small displacement와 linear material 가정이 적절한지 확인합니다.
  • 필요 output이 field인지 history인지 step에서 요청합니다.
5. Load / BC

하중과 경계조건

모델 신뢰도를 가장 크게 좌우합니다. force, pressure, displacement, fixed support는 서로 다른 물리 질문입니다.

  • 반력 합이 외력과 맞는지 확인할 수 있도록 BC를 설계합니다.
  • point load와 fully fixed edge는 local singularity를 만들 수 있습니다.
6. Mesh / Job / Visualization

계산과 증거 읽기

mesh를 만들고 job을 실행한 뒤, visualization에서 결과를 확인합니다. 하지만 visualization은 마지막 단계이고, job evidence가 먼저입니다.

  • mesh refinement별 trend를 비교합니다.
  • job message, warning, reaction balance를 결과 그림보다 먼저 확인합니다.
Input Deck Literacy

`.inp` 파일은 Abaqus가 실제로 읽은 해석 모델입니다. CAE 화면에서 무엇을 했든, 최종 진실은 input deck에 남습니다. 팀장은 아래 키워드가 어떤 모델 가정을 뜻하는지 읽을 수 있으면 됩니다.

*Material, name=AL6061

Geometry / Mesh

*Node, *Element

해석 자유도와 mesh topology를 정의합니다. element type은 수치 안정성과 stress 해석 방식에 직접 연결됩니다.

  • C3D8R: 8-node reduced integration solid.
  • S4R: 4-node reduced integration shell.
  • B31: 2-node beam element.
Material

*Material, *Elastic

선형 정적 기본 모델에서는 E와 nu가 핵심입니다. 물성 출처, 단위, 온도 범위를 확인해야 합니다.

  • 단위계는 Abaqus가 자동 판단하지 않습니다.
  • MPa-mm-tonne 계열과 Pa-m-kg 계열 혼동을 조심합니다.
Step / Load

*Step, *Static, *Boundary, *Cload

무엇을 풀었는지, 어디를 잡았는지, 어떤 하중을 줬는지의 진실입니다. 결과 이상은 대부분 여기에서 시작됩니다.

  • Fixed face가 실제 fixture를 대표하는지 확인합니다.
  • Force와 prescribed displacement의 의미를 구분합니다.
Output

*Output, Field / History

나중에 볼 수 있는 증거는 요청한 output으로 제한됩니다. 필요한 RF, U, S, energy/history가 요청되었는지 확인합니다.

  • field output: 공간 분포 contour.
  • history output: 특정 node/RP/region의 시간 또는 increment 이력.
ODB and Job Evidence

`.odb`는 결과 파일이지만, 숫자와 contour만 보는 곳이 아닙니다. 반력 균형, 변형 형상, field/history output, warning과 연결해서 읽어야 합니다.

Reaction Balance

RF 합계와 외력 합계

선형 정적 모델의 첫 검증은 반력 합이 적용 하중과 균형을 이루는지 확인하는 것입니다.

  • 고정 지점의 RF1/RF2/RF3를 합산합니다.
  • 적용한 force, pressure resultant와 크기/방향이 맞는지 봅니다.
  • 맞지 않으면 하중 누락, 단위 오류, BC 오류, contact 누락 가능성을 의심합니다.
팀장 기준: contour plot을 보기 전에 RF balance가 설명되어야 합니다.
Deformed Shape

변형 형상의 물리성

최대 응력보다 먼저 전체 변형 형상이 상식과 맞는지 확인합니다. 방향이 이상하면 좌표계, 하중, 구속이 잘못되었을 수 있습니다.

  • deformation scale factor가 과장되어 있음을 인식합니다.
  • 하중 방향과 변위 방향이 일관적인지 확인합니다.
  • 구속면 주변에 비현실적 국부 변형이 없는지 봅니다.
Stress Readout

S, Mises, Principal

결과 stress는 재료와 failure mode에 맞춰 골라야 합니다. 금속 항복은 Mises, 취성 균열은 Max Principal을 우선합니다.

  • integration point, nodal averaging, element averaging 차이를 확인합니다.
  • sharp corner peak를 설계 판단값으로 쓰지 않습니다.
  • mesh refinement로 stress trend를 확인합니다.
Job Files

.msg, .sta, .dat, .log

해석이 정상 종료되었다는 말과 신뢰 가능하다는 말은 다릅니다. job 파일은 solver가 남긴 정직한 흔적입니다.

  • .sta: step/increment 진행과 종료 상태.
  • .msg: warning, convergence, contact diagnostics.
  • .dat/.log: 출력 요청, 실행 정보, 에러/경고 흔적.
팀장 기준: warning을 "있었지만 무시"가 아니라 "결과에 영향 없음"으로 설명해야 합니다.
Two-Week Work Plan
Day 1

Abaqus/CAE 전체 map 잡기

Part, Property, Assembly, Step, Load, Mesh, Job, Visualization이 각각 어떤 해석 가정을 만드는지 한 장으로 정리합니다.

Day 2

Cantilever 선형 정적 모델 만들기

손계산 tip displacement와 Abaqus U 결과를 비교하고, RF balance를 확인합니다.

Day 3

Input deck 읽기

CAE에서 만든 모델을 `.inp`로 export하고, *Node, *Element, *Material, *Boundary, *Cload, *Output을 표시합니다.

Day 4

ODB 결과와 field/history output 비교

U, RF, S, Mises를 보고, contour와 XY/history 데이터가 각각 어떤 질문에 답하는지 구분합니다.

Day 5

Mesh refinement 1차 비교

cantilever mesh를 3단계로 나누고, tip displacement와 peak stress가 어떻게 수렴하는지 기록합니다.

Day 6

Plate with hole 모델

stress concentration 위치와 Mises/Max Principal 차이를 확인하고, hole 주변 mesh refinement가 결과에 미치는 영향을 봅니다.

Day 7

Simple bracket 모델

fixture, load introduction, stress peak 위치를 보고 실제 구조물에서 어떤 단순화가 위험한지 정리합니다.

Day 8

Job message review

정상 종료, warning, zero pivot, excessive distortion, singularity candidate 같은 메시지를 분류합니다.

Day 9

팀원 모델 하나를 evidence 관점으로 읽기

실무자가 만든 기존 모델에서 `.inp` 구조, output request, RF balance, mesh sensitivity 여부를 확인합니다.

Day 10

Abaqus Evidence Memo 작성

결과 이미지 1장, RF balance 1개, input deck map 1개, warning 해석 1개를 포함한 팀장용 memo를 작성합니다.

Deliverables
Deliverable 1

Abaqus Workflow Map

CAE module별로 어떤 물리/수치 가정이 들어가는지 한 장으로 정리합니다.

  • Part: geometry idealization.
  • Property: material/section assumption.
  • Step/Load/BC: physics setup.
  • Job/Visualization: solve evidence와 interpretation.
Deliverable 2

Input Deck Map

`.inp` 한 개를 열고 핵심 keyword를 색칠하듯 분해합니다. 목적은 deck을 직접 작성하는 것이 아니라, 모델의 진실을 읽는 것입니다.

  • *Material과 단위계 확인.
  • *Boundary와 *Cload의 실제 의미 확인.
  • *Output 요청이 리뷰에 충분한지 확인.
Deliverable 3

Mesh Sensitivity Note

cantilever, plate with hole, bracket 중 2개를 골라 mesh refinement trend를 기록합니다.

  • 관심 결과: U, Mises, Max Principal 중 명시.
  • 수렴하는 값과 수렴하지 않는 peak를 분리.
  • 의사결정 가능 여부를 한 문장으로 결론.
Deliverable 4

Abaqus Evidence Memo

팀원이 결과를 가져왔을 때 확인할 최소 evidence 형식을 만듭니다.

  • 해석 목적과 failure mode.
  • 모델 가정: element, material, BC, load.
  • 증거: RF balance, deformation, mesh trend, warning status.
  • 결론: 지금 결정 가능한 것과 보류해야 할 것.
Team Review Questions
Model Setup

모델 구성 질문

  • 이 geometry simplification은 관심 stress path를 바꾸지 않는가?
  • 이 material property는 실제 온도와 공정 조건에서 유효한가?
  • 이 BC는 실제 fixture를 대표하는가, 아니면 편의를 위한 완전 구속인가?
Solve Evidence

해석 증거 질문

  • RF 합계와 외력 합계가 맞는가?
  • warning이 결과 해석에 영향 없다고 말할 근거가 있는가?
  • field output과 history output 중 어느 것이 이 의사결정에 더 적합한가?
Decision

의사결정 질문

  • 현재 mesh로 말할 수 있는 결론과 말하면 안 되는 결론은 무엇인가?
  • peak stress가 아니라 trend로 판단해야 하는 항목은 무엇인가?
  • 다음 해석 하나만 한다면 uncertainty를 가장 줄이는 실험은 무엇인가?
