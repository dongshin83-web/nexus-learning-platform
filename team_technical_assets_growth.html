Shared Growth

기술자산화는 팀이 함께 키우는 성장 인프라다

좋은 자산은 완성된 보고서가 아니라 다음 사람이 문제를 더 빨리 이해하고, 모델과 조건을 재사용하며, 필요한 시점에 판단 근거를 만들 수 있게 하는 학습 재료입니다.

Use Case 신규 인원 온보딩

대표 BP를 읽고, 모델과 조건을 재현하며, 판단 기준을 토론하는 방식으로 기술 맥락을 익힙니다.

Use Case 유사 과제 착수

새 과제 시작 전에 유사한 기술영역과 연계 태그를 검색해 기존 판단 기준을 먼저 확인합니다.

운영 Loop

대표 사례가 개인 문서에 머물지 않고 팀 학습 자산으로 돌아오게 하는 최소 흐름입니다.

01 Select 대표 사례 선별

모든 대응을 올리지 않고 반복성, 의사결정 영향, 교육 활용성이 있는 사례부터 고릅니다.

02 Decompose 재사용 요소 분해

문제, 모델, 물성, 경계조건, 판단 기준, 한계/주의점을 나누어 남깁니다.

03 Review 함께 검토

Reviewer와 파트리더가 기술 타당성, 재사용성, 교육 활용성을 점검합니다.

04 Link 내부 자료 연결

문서, 모델 폴더, 결과표, 검토 코멘트 링크를 한 카드에서 찾을 수 있게 묶습니다.

05 Reuse 학습과 재사용

신규 인원 교육, 유사 과제 착수, 기술 Map 보완에 다시 활용합니다.

카드 작성 기준

프로토타입을 직접 수정할 때는 카드 하나가 하나의 자산 단위가 되도록 유지합니다.

문제보다 판단 질문을 쓴다

무엇을 해석했는지보다 어떤 의사결정에 필요한 판단 근거였는지를 먼저 적습니다.

결과와 입력값을 분리한다

모델, 물성, 경계조건, 실험 피드백, 한계/주의점을 따로 남겨야 재사용할 수 있습니다.

기술영역은 하나만 고른다

기본 분류는 하나로 두고, AI/타 Domain/신뢰성/공정 연계는 태그로 여러 개 붙입니다.

링크는 내부망에서 교체한다

현재 링크는 placeholder입니다. 실제 운영에서는 회사 내부 문서와 모델 폴더 주소로 바꿉니다.
