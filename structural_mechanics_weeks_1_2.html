구조역학과 FEA 판단 언어 정렬

12-Week Plan

첫 2주는 Abaqus 버튼을 외우기 전에, 해석 모델을 같은 언어로 읽기 위한 물리/수치 해석 감각을 맞추는 구간입니다. 목표는 stress contour를 해석하는 것이 아니라, 왜 그런 stress가 생겼는지 load path, element, boundary condition, mesh sensitivity 관점에서 설명하는 것입니다.

Governing Ideas

응력: 하중이 단면에 남기는 세기

σ=FA
stress = force / area

응력은 단순한 색깔값이 아니라 하중이 단면을 통해 전달되는 방식입니다. 확인 기준은 "이 힘은 실제로 어디로 흘러가는가?"입니다.

변형률: 늘어난 양을 원래 길이로 나눈 값

ε=ΔLL
strain = length change / original length

변형률은 재료가 국부적으로 얼마나 늘거나 줄었는지 보는 언어입니다. MD의 원자 변위 감각을 연속체 스케일로 올린 값입니다.

선형 탄성: 응력과 변형률의 비례 관계

σ=Eε
stress = elastic modulus x strain

이 관계가 깨지는 순간 plasticity, damage, nonlinear material 질문이 시작됩니다.

FEA 균형식: 강성, 변위, 하중의 관계

Ku=f
stiffness matrix x displacement vector = force vector

stiffness, displacement, load의 관계를 보면 solver 결과를 그림이 아니라 방정식으로 읽게 됩니다.

Result Language

구조해석 결과는 보통 stress, displacement, stiffness 세 가지 언어로 정리됩니다. Abaqus에서는 stress와 displacement가 직접 output variable로 나오지만, stiffness는 대개 반력과 변위로 계산하는 derived result입니다.

Stress

Stress는 "어디가 위험한가"의 언어

하중이 구조 내부에서 어떤 방향과 크기로 전달되는지를 나타냅니다. 재료의 failure mode와 비교해야 의미가 생깁니다.

  • Abaqus 기본 변수: S, 전체 stress tensor.
  • 구성 성분: S11, S22, S33, S12, S13, S23.
  • Principal stress는 SP, stress invariant는 SINV 또는 S의 invariant로 확인합니다.
보고서 기본값: ductile metal은 Mises, brittle/tension crack은 Max Principal, 방향성 검토는 S11/S22 같은 component.
Displacement

Displacement는 "얼마나 움직였는가"의 언어

구조물이 하중을 받은 뒤 node가 이동한 양입니다. 기능 요구사항, clearance, 조립 간섭, 변형 허용치와 직접 연결됩니다.

  • Abaqus 기본 변수: U, nodal displacement.
  • 구성 성분: U1, U2, U3, 그리고 magnitude인 U, Magnitude.
  • beam/shell 회전 자유도가 중요하면 UR1, UR2, UR3도 같이 봅니다.
보고서 기본값: 요구사항 방향이 있으면 해당 component, 전체 처짐이면 U magnitude와 deformed shape.
Stiffness

Stiffness는 "얼마나 버티는가"의 언어

하중을 줬을 때 변위가 얼마나 생기는지의 비율입니다. Abaqus contour에 바로 나오는 일반 결과라기보다, 반력과 변위 history에서 계산합니다.

  • 선형: k = F / delta.
  • 비선형 secant stiffness: 특정 하중점에서 F / delta.
  • 비선형 tangent stiffness: load-displacement curve의 국부 기울기 dF / ddelta.
보고서 기본값: 기준 node 또는 reference point의 U와 support/prescribed point의 RF로 load-displacement curve를 만들고 slope를 계산.
Abaqus Output Choice

보고서의 대표 응력은 failure mode와 재료 모델에 맞춰 선택해야 합니다. Maximum Principal이 익숙하더라도, ductile metal yielding에는 Mises가 더 자연스러운 기본값입니다.

Mises

Von Mises Equivalent Stress

3D stress tensor를 ductile yielding 판단용 scalar로 줄인 값입니다.

  • 추천: ductile metal, plasticity, bracket/frame 일반 강도 검토.
  • 이유: hydrostatic pressure보다 distortion energy가 항복을 지배한다는 가정과 맞습니다.
  • Abaqus 이름: MISES, MISESONLY, shell/beam section envelope에는 MISESMAX.
Max Principal

Maximum Principal Stress

한 점에서 가능한 모든 면을 돌려봤을 때 가장 큰 인장 normal stress입니다.

  • 추천: brittle material, glass/ceramic, crack opening, adhesive peel, tensile failure.
  • 이유: 균열과 취성 파손은 최대 인장 주응력에 민감한 경우가 많습니다.
  • Abaqus 이름: SP for all principal stresses, SP3 for maximum principal stress history output.
Min Principal

Minimum Principal Stress

가장 큰 압축 방향의 principal stress입니다. 압축, crushing, contact bearing 문제에서 봅니다.

  • 추천: compressive crushing, bearing/contact pressure 주변, 취성재 압축 검토.
  • 주의: 부호 convention과 좌표계를 확인해야 합니다.
  • Abaqus 이름: SP1 for minimum principal stress history output.
Tresca

Tresca / Maximum Shear 기준

principal stress 차이로부터 최대 전단 지배를 보는 값입니다.

  • 추천: shear failure, shaft/torsion, 보수적 금속 항복 검토.
  • 일반 금속 설계 보고에서는 Mises와 함께 비교값으로 두는 편이 읽기 쉽습니다.
  • Abaqus 이름: TRESC.
Pressure

Hydrostatic Pressure Stress

응력 tensor의 평균 normal stress 성분입니다. 체적 압축/팽창, gasket, porous material에서 의미가 큽니다.

  • 추천: hydrostatic compression, gasket/seal, porous material, void growth risk.
  • 주의: 일반 강도 contour의 대표값으로 내보내면 사용자가 오해하기 쉽습니다.
  • Abaqus 이름: PRESS 또는 PRESSONLY.
Components

S11, S22, S33, S12, S13, S23

좌표계 방향의 normal/shear stress입니다. 방향 자체가 의사결정 기준일 때 씁니다.

  • 추천: composite, anisotropic material, 특정 방향 허용응력 검토.
  • 주의: 좌표계가 바뀌면 의미도 바뀝니다.
  • Abaqus 이름: S for all components, Sij for individual history components.
Report Rule
Primary Report Set

기본 보고 묶음

결과 한 장에는 응력 하나만 넣지 말고, displacement와 evidence를 함께 둬야 합니다.

  • Ductile metal: Mises max/region trend + U magnitude 또는 요구 방향 U component + RF balance.
  • Brittle/tension: Max Principal + crack/opening 방향 component + mesh sensitivity.
  • Stiffness: RF-U curve에서 secant 또는 tangent stiffness를 명시합니다.
Peak Rule

최대값 하나만 보고하지 않기

Peak stress는 singularity, sharp corner, point load, constraint 때문에 mesh를 조일수록 무한히 커질 수 있습니다.

  • 최대값은 위치, averaging 여부, element/integration point 여부를 함께 적습니다.
  • hot spot 주변 path/area average와 mesh refinement trend를 같이 봅니다.
  • 의사결정에는 peak보다 trend, margin, correlation evidence가 더 중요할 수 있습니다.
Two-Week Work Plan
Week 1

판단 언어 정렬

stress, strain, stiffness, load path, element, DOF를 같은 의미로 말할 수 있게 만듭니다.

  • normal/shear stress, principal stress, von Mises stress 용도 구분.
  • beam/shell/solid element 선택 기준 정리.
  • BC와 load가 실제 load path를 바꾸는 사례 수집.
Week 2

Benchmark로 검증 감각 만들기

간단한 benchmark를 통해 손계산, mesh sensitivity, Abaqus output을 연결합니다.

  • cantilever: tip displacement와 reaction balance 비교.
  • plate with hole: stress concentration과 mesh sensitivity 확인.
  • simple bracket: Mises, Max Principal, displacement를 함께 보고서화.
Deliverable

해석 검토 체크리스트

결과 검토에 바로 쓸 확인 항목을 남깁니다.

  • 이 모델이 현실을 가장 크게 단순화한 지점은 어디인가?
  • 이 stress peak는 failure risk인가, singularity인가?
  • 현재 mesh로 의사결정 가능한 결과와 불가능한 결과는 무엇인가?
